Developer Field Notes

Git vs GitHub

Git is a version control tool that runs locally on your computer. It tracks changes to your files and lets you save snapshots called commits.

GitHub is a cloud platform that hosts Git repositories. It adds collaboration features like pull requests, issues, and GitHub Pages.

Think of Git as the tool and GitHub as the place where you share your work.

The daily loop

  1. Edit — make changes to your files
  2. Stage — run git add to prepare changes
  3. Commit — run git commit to save a snapshot
  4. Push — run git push to send commits to GitHub

Git command reference

  • git init — initialize a new local repository
  • git status — show the state of your working directory and staging area
  • git add <file> — stage a specific file
  • git add . — stage all changed files at once
  • git commit -m "message" — save a snapshot of staged changes
  • git log --oneline — view commit history in compact format
  • git diff — show unstaged changes line by line
  • git push — send local commits to the remote repository
  • git pull — fetch and merge changes from the remote
  • git checkout -b <name> — create and switch to a new branch
  • git checkout <name> — switch to an existing branch
  • git remote -v — list connected remote repositories

Do Not Commit

  • .env files
  • node_modules/
  • API keys or secrets
  • Passwords or credentials
